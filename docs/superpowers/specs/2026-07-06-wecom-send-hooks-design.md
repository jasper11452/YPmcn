# 企微发送 Hook 设计

## 目标

将企微发送固定为：用户确认 → 调用 `send_wx_messages.py` → 成功后创建一次性提醒 → 停止并等待用户继续。

## 识别与参数

- 仅匹配 `exec`/`bash`/`shell` 工具中把 `send_wx_messages.py` 作为独立命令参数的调用，避免普通文本误命中。
- 命令必须包含 `--remind-at <ISO 8601>`，且时间有效并晚于当前时间。
- 缺少会话标识、提醒时间或 Cron 服务时，在发送前阻断。

## Hook 流程

1. `before_tool_call`：返回 `requireApproval`；只允许 `allow-once` 或 `deny`，超时默认拒绝。
2. `after_tool_call`：仅在脚本无错误且退出码为 0 时创建一次性 Cron `agentTurn`，投递到原会话。
3. 发送成功后立即把该会话标记为等待用户；等待期间，`before_tool_call` 阻断后续工具调用。
4. 定时任务到点只向用户发送提醒，不执行下一业务步骤。
5. `message_received` 收到该会话的新用户消息后解除等待；其他系统、Cron 或 Agent 事件不得解除。

## 失败处理

- 用户拒绝或确认超时：脚本不执行。
- 脚本失败：不创建提醒、不进入等待态。
- Cron 创建失败：保持等待态并暴露明确错误，禁止自动推进。
- 同一成功发送只创建一个提醒任务，以 `toolCallId` 去重。

## 验证

测试覆盖：精确识别、发送前审批、拒绝/超时、无效提醒时间、成功后建任务、失败不建任务、等待态阻断、仅用户新消息解除、重复结果不重复建任务。
