## 当前 MVP 口径

- 下游工具使用链式业务 ID，不使用旧的 `demand_id + demand_version` 作为主入参。
- Agent 先把 Brief 解析成 schema 内 JSON。必填项为 `platform`、`submission_deadline_at`、`raw_messages_json`、`budget_min_cents`、`budget_max_cents`、`budget_raw`、`rebate_min_rate`、`rebate_max_rate`、`rebate_raw`、`quantity_total`；单价/预算和返点都按区间表达。
- 必填项缺失时不可继续。必填项满足后，额外需求要参考 `YPmcn/skills/media-assistant/references/creator_candidate_pool_schema.csv` 匹配到需求表字段或筛选字段，并让用户确认。
- `search_creators` 根据需求表主键读取非空字段，与达人表同字段匹配筛选。
- `rank_mcns` 根据候选池主键读取候选和 MCN 数据，计算达人供需关系、建议手扒比例，并拟出建议询价 MCN 列表。
- 用户确认或修改 MCN 列表后，Agent 根据需求表主键读取非空需求字段，拟写企微询价消息给用户确认。
- 企微发送接口字段固定，不修改接口。每个 MCN 必须有唯一填报链接，并把候选池中属于该 MCN 的达人预填到对应填报表作为推荐。
- 需要手扒时，同步启动手扒程序；机构回填和手扒达人最终都回收到候选池，统一去重、筛选、精排。
- Hooks 只做流程顺序、链式 ID、企微发送、权限、等待锁和风险字段等硬护栏；链式 `id` 映射旧业务表主键，不代表新表名。
- Hooks 不因为缺少 `trace_id`、响应信封细项或语义启发式疑点阻断主流程。
- `create_with_distributions` 必须用 MCP 工具调用，不允许 Bash/curl 直连。
- 分发成功后必须停，等用户确认是否进入精排。
