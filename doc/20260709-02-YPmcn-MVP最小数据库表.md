# YPmcn MVP 最小数据库表

> 更新日期：2026-07-09  
> 目标：在不改旧表名、不改旧工具名的前提下，定义“先跑通”的最小数据闭环。

## 1. 结论

MVP 不新增另一套泛化表名。正式表名沿用原数据库设计：

```text
customer_demands
xhs_creator_accounts
dy_creator_accounts
creator_supply_offers
creator_candidate_pool
mcn_agencies
mcn_recommendation_items
mcn_inquiries
mcn_submission_items
recommendation_runs
creator_recommendation_items
creator_submissions
submission_batches
```

最小链路关系：

```text
customer_demands
  -> creator_candidate_pool
      -> mcn_recommendation_items
          -> mcn_inquiries
              -> mcn_submission_items
      -> recommendation_runs
          -> creator_recommendation_items
              -> creator_submissions
                  -> submission_batches

xhs_creator_accounts / dy_creator_accounts
  -> creator_supply_offers
      -> creator_candidate_pool

mcn_agencies
  -> creator_supply_offers
  -> mcn_recommendation_items
  -> mcn_inquiries
```

工具名也沿用既有 MCP 工具名：`validate_requirement`、`search_creators`、`rank_mcns`、`create_with_distributions`、`ingest_mcn_submissions`、`manual_source_creators`、`rank_creators`、`create_submission_batch`、`record_client_feedback`、`audit_manual_adjustment`、`get_creator_detail`、`get_recommendation_run_detail`。

## 2. 需求主表：`customer_demands`

用途：`validate_requirement` 写入客户 Brief 的结构化结果，是整个链路入口。

字段来源：以 `YPmcn/skills/media-assistant/references/creator_candidate_pool_schema.csv` 为准。MVP 中，`validate_requirement` 写入的需求主表就是该 CSV 的字段字典；Agent 解析出的确定字段直接写入 `customer_demands` 同名字段，额外需求也必须先对照该 CSV 的字段能力做匹配和确认。

CSV 解析规则：

- 第一行 `序号,字段,抖音是否存在,小红书是否存在,合并结果,抖音说明,小红书说明,备注` 是字段字典元信息，不是数据库列。
- 数据库列取 CSV 的 `字段` 列；`id(32位uuid)` 入库列名规范化为 `id`。
- 本次读取到 `字段` 列共 153 个字段，其中抖音字段 102 个、小红书字段 101 个、两平台共同字段 72 个。
- `customer_demands` 以这 153 个字段为可写字段全集；未解析或未确认的字段保持空值。
- 平台达人表只继承对应平台存在的字段，不另造字段名。

必要系统字段：

| 字段 | 说明 |
|---|---|
| `id` | MCP 返回给工具链使用的需求主键，可映射到旧 `demand_id` |
| `demand_id` | 稳定需求业务 ID |
| `demand_version` | 需求版本 |
| `status` | `draft/ready/screening/inquiring/submitted/closed/archived` |
| `created_at` / `updated_at` | 时间 |

当前继续前必填业务字段：

```text
platform
submission_deadline_at
raw_messages_json
budget_min_cents
budget_max_cents
budget_raw
rebate_min_rate
rebate_max_rate
rebate_raw
quantity_total
```

区间规则：

- 预算/单价写 `budget_min_cents` + `budget_max_cents`。
- 返点写 `rebate_min_rate` + `rebate_max_rate`。
- 单值写成闭区间。
- 只有上限时，下限按业务可接受下界归一，并保留 `budget_raw` / `rebate_raw`。

额外需求规则：

- 能匹配到 CSV 字段的，写入 `customer_demands` 同名字段。
- 不能确定字段或口径的，先保留在 `raw_messages_json` / `note`，并让用户确认后再写入同名字段。
- 用户未确认额外字段映射前，不进入 `search_creators`。

## 3. 达人资源库：`xhs_creator_accounts` / `dy_creator_accounts`

用途：平台达人账号资源库。物理表按平台拆分，不合并成统一达人主表。

字段来源：两张平台账号表从 `customer_demands` 的 CSV 字段字典继承字段。共同字段保持同名，平台专属字段留在对应平台表；不得把 CSV 字段重命名成另一套账号字段。

### 3.1 `xhs_creator_accounts`

必要字段：

| 字段 | 说明 |
|---|---|
| `kw_uid` | 小红书达人唯一账号标识 |
| `platform` | 固定 `xhs` |
| `nickname` | 昵称 |
| `organization` | MCN/机构名称 |
| `followercount` | 粉丝数 |
| `content_type_label` / `content_tag` / `kol_persona_label` / `content_feature_label` | 内容标签 |
| `view_medium` / `avginteract` | 内容表现 |
| `cpe_l1` / `cpm_l1` | 效率指标 |
| `kol_official_price_l1` / `kol_predict_price_l1` | 参考报价 |
| `date` | 数据日期 |

### 3.2 `dy_creator_accounts`

必要字段：

| 字段 | 说明 |
|---|---|
| `kw_uid` | 抖音达人唯一账号标识 |
| `platform` | 固定 `dy` |
| `nickname` | 昵称 |
| `organization` | MCN/机构名称 |
| `followercount` | 粉丝数 |
| `content_type_label` / `content_theme_label` / `industry_tag_label` | 内容标签 |
| `view_medium` / `avginteract` | 内容表现 |
| `cpe_l1` / `cpm_l1` | 效率指标 |
| `kol_official_price_l1` / `kol_predict_price_l1` | 参考报价 |
| `date` | 数据日期 |

关键约束：

```text
xhs_creator_accounts: unique(kw_uid)
dy_creator_accounts: unique(kw_uid)
跨平台引用统一使用 platform + kw_uid
```

## 4. 供给和机构：`creator_supply_offers` / `mcn_agencies`

### 4.1 `creator_supply_offers`

用途：连接平台达人账号、报价、返点、机构关系。预算筛选和 MCN 聚合读取这张表，不直接读取平台账号表里的参考价作为最终报价。

必要字段：

| 字段 | 说明 |
|---|---|
| `offer_id` | 供给记录 ID |
| `platform` | `xhs/dy` |
| `platform_account_id` / `kw_uid` | 平台账号标识 |
| `mcn_id` | 机构 ID |
| `price_cents` | 本次可用报价，单位分 |
| `rebate_rate` | 本次返点 |
| `content_format` | 内容形式 |
| `valid_until` | 报价有效期 |
| `source_channel` | `rate_card/mcn_returned/manual_search` 等 |
| `created_at` / `updated_at` | 时间 |

### 4.2 `mcn_agencies`

用途：MCN/供应商事实表，支持排序、企微发送和机构画像。

必要字段：

| 字段 | 说明 |
|---|---|
| `mcn_id` | MCN/供应商 ID |
| `agency_name` | 机构正式名称 |
| `backend_name` | 后台简称/别名 |
| `primary_platforms` | 机构主要平台 |
| `account_types` | 机构擅长账号类型 |
| `default_rebate_rate` | 默认返点 |
| `policy_rating` | 政策评级 |
| `response_rate` / `valid_submission_rate` / `selected_rate` | 履约质量指标 |
| `platform_profiles_json` | 分平台返点、评级与履约画像 |
| `contact_owner` / `wecom_group_id` | 内部对接人和企微群 |
| `cooperation_status` | 合作状态 |
| `notes` | 备注 |
| `created_at` / `updated_at` | 时间 |

## 5. 候选池：`creator_candidate_pool`

用途：`search_creators` 写入的候选池，也是 MCN 回填和手扒达人最终汇总的中间层。

必要字段：

| 字段 | 说明 |
|---|---|
| `candidate_id` | 候选记录 ID |
| `demand_id` / `demand_version` | 需求版本 |
| `platform` | `xhs/dy` |
| `kw_uid` / `platform_account_id` | 达人账号 |
| `offer_id` | 供给报价记录 |
| `candidate_source` | `rate_card/mcn_returned/manual_search/similar_creator/historical_reuse` |
| `mcn_id` | 所属机构，可空 |
| `price_cents` | 本次候选报价 |
| `rebate_rate` | 本次返点 |
| `matched_fields_json` | 命中字段 |
| `missing_fields_json` | 缺失字段 |
| `score_snapshot_json` | 初筛/软匹配分数 |
| `source_detail_json` | 手扒或回填来源 |
| `created_at` | 时间 |

关键约束：

```text
unique(demand_id, demand_version, platform, kw_uid, candidate_source)
```

## 6. MCN 排序与询价

### 6.1 `mcn_recommendation_items`

用途：`rank_mcns` 输出的 MCN 排序结果，包含供需关系、建议手扒比例和建议询价列表。

必要字段：

| 字段 | 说明 |
|---|---|
| `mcn_recommendation_id` | MCN 推荐记录 ID |
| `demand_id` / `demand_version` | 需求版本 |
| `mcn_id` | MCN ID |
| `platform` | 平台 |
| `candidate_count` | 当前 MCN 可供达人数量 |
| `supply_ratio` | 当前供需倍数 |
| `manual_sourcing_ratio` | 建议手扒比例 |
| `rank_score` | MCN 排序分 |
| `recommend_reason` | 推荐原因 |
| `formula_snapshot_json` | 计算快照 |
| `created_at` | 时间 |

### 6.2 `mcn_inquiries`

用途：`create_with_distributions` 成功后的询价事实。每个 MCN 有唯一填报链接。

必要字段：

| 字段 | 说明 |
|---|---|
| `inquiry_id` | 询价 ID |
| `demand_id` / `demand_version` | 需求版本 |
| `mcn_id` | MCN ID |
| `supplier_id` | 企微供应商 ID |
| `deadline` / `remind_at` | 截止/提醒时间 |
| `usage_scope` | 固定 `project` |
| `form_fields_json` | 表单字段 |
| `wecom_message_json` | 发送消息 |
| `prefill_rows_json` | 当前 MCN 预填达人 |
| `fill_link` / `token` | 唯一填报链接或 token |
| `response_status` | `sent/returned/failed/cancelled` |
| `sent_at` / `returned_at` | 时间 |

`prefill_rows_json` 只放 `creator_candidate_pool` 中属于当前 MCN 的候选达人，作为机构打开填报表时的推荐底稿。

### 6.3 `mcn_submission_items`

用途：机构回填达人明细。`ingest_mcn_submissions` 标准化回填后，符合条件的达人写回 `creator_candidate_pool(candidate_source=mcn_returned)`。

必要字段：

| 字段 | 说明 |
|---|---|
| `submission_item_id` | 回填明细 ID |
| `inquiry_id` | 询价 ID |
| `mcn_id` | MCN ID |
| `platform` | 平台 |
| `kw_uid` / `platform_account_id` | 达人账号 |
| `submitted_price_cents` | 机构回填报价 |
| `submitted_rebate_rate` | 机构回填返点 |
| `schedule_json` | 档期 |
| `validity_status` | `accepted/rejected/need_review` |
| `candidate_id` | 写回候选池后的 ID |
| `raw_payload_json` | 原始回填 |

## 8. 工具和表的对应关系

| 工具 | 读 | 写 |
|---|---|---|
| `validate_requirement` | 客户 Brief / 旧需求版本 | `customer_demands` |
| `search_creators` | `customer_demands`, `xhs_creator_accounts`, `dy_creator_accounts`, `creator_supply_offers` | `creator_candidate_pool` |
| `rank_mcns` | `creator_candidate_pool`, `mcn_agencies`, `creator_supply_offers` | `mcn_recommendation_items` |
| `create_with_distributions` | `mcn_recommendation_items`, `mcn_agencies`, `creator_candidate_pool` | `mcn_inquiries` |
| `ingest_mcn_submissions` | `mcn_inquiries`, 机构回填源 | `mcn_submission_items`, `creator_supply_offers`, `creator_candidate_pool` |
| `manual_source_creators` | `customer_demands`, 手扒源 | `creator_candidate_pool`, 可更新平台账号表和 `creator_supply_offers` |
| `rank_creators` | `customer_demands`, `creator_candidate_pool` | `recommendation_runs`, `creator_recommendation_items` |
| `create_submission_batch` | `recommendation_runs`, `creator_recommendation_items` | `submission_batches`, `creator_submissions` |
| `record_client_feedback` | `submission_batches`, `creator_submissions` | 更新 `creator_submissions.client_feedback_status`，必要时触发重排/改需求 |
| `audit_manual_adjustment` | 推荐/提报明细 | 写入对应 JSON 调整日志，不新增独立审计表 |

## 9. MVP 明确不改

| 不改 | 原因 |
|---|---|
| 不重命名 `customer_demands` | 避免破坏既有需求版本语义 |
| 不合并 `xhs_creator_accounts` / `dy_creator_accounts` | 原设计明确达人账号按平台拆物理表 |
| 不拆分 `creator_candidate_pool` | 旧候选池已承接粗筛、软匹配、手扒、MCN 回填 |
| 不重命名 MCP 工具 | Agent、hooks、MCP provider 和测试都按旧工具名协作 |
| 不新增手扒任务表 | 手扒详情写 `source_detail_json`，工具 `manual_source_creators` 不创建任务实体 |
